# Product Principles

## Purpose

This document defines the immutable principles that guide every product and technical decision inside MOLT OS.

Every feature, workflow, AI capability and interface should comply with these principles.

If a proposed feature conflicts with them, the principles take precedence.

---

# Stable Principles

These principles should change only in exceptional situations.

## 1. Human First

Technology exists to help people.

People should never adapt their thinking to fit the software.

The software must adapt to the way people naturally think and work.

---

## 2. Reduce Cognitive Load

The purpose of MOLT OS is not to store information.

Its purpose is to reduce the amount of information the user has to remember.

Every new feature should reduce mental effort.

Never increase complexity without creating significantly more value.

---

## 3. AI Assists. Human Decides.

Artificial Intelligence never owns the final decision.

AI may:

- recommend
- organize
- summarize
- predict
- prepare

The human always confirms important actions.

---

## 4. Context Before Questions

Never ask the user something the system already knows.

If the current project is known,
use it.

If today's schedule is known,
use it.

If the active timer is running,
use it.

Context always comes first.

---

## 5. One Source of Truth

Every piece of information has exactly one owner.

Projects own project data.

Calendar owns scheduled events.

Personal Items own personal responsibilities.

Knowledge owns reference information.

Never duplicate information.

---

## 6. AI First

Everything that can naturally be achieved through conversation should be possible through the assistant.

The graphical interface exists for visibility and control.

The assistant exists for speed.

---

## 7. Keyboard First

Every frequently used action should be available through:

Keyboard

Command Palette

Voice

without unnecessary mouse interaction.

---

## 8. Voice Equals Typing

Voice commands are not a secondary feature.

The assistant should understand spoken language exactly as written language.

---

## 9. Work and Life Together

MOLT OS manages the person.

Not only the business.

Projects and personal life coexist inside one trusted system while remaining logically separated.

---

## 10. Calm Software

The interface should create clarity.

Never noise.

Avoid unnecessary notifications.

Avoid visual overload.

Avoid modal dialogs whenever possible.

Speed and calmness are product features.

---

## 11. Performance Is a Feature

Slow software increases cognitive load.

Responsiveness is part of the user experience.

Every interaction should feel immediate.

---

## 12. Explainability

When AI recommends something important,
the user should understand why.

Recommendations should be transparent whenever possible.

---
